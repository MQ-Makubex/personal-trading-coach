# 个人交易教练台与模式档案设计

- 日期：2026-07-11
- 状态：已确认，待实施计划
- 范围：首页教练台、交易底账时间筛选、股票周期故事、交易模式档案、纪律消息存储

## 1. 目标

个人站的核心目标不是展示更多行情或放大账户盈亏，而是持续打磨一套适合用户自己的交易模式。

本次设计需要让用户打开站点后依次得到：

1. 当前账户发生了什么。
2. 当前是否通过纪律资格检查。
3. 当前有哪些交易模式正在验证。
4. 今日预案、股票池和持仓需要处理什么。
5. 全历史交易结果反映了哪些能力结构。

## 2. 非目标

- 不生成交易评分、能力总分或高/中/低评级。
- 不预测涨跌，不把资格状态表达成买卖信号。
- 不自动升级模式为“可复制”。
- 不回溯篡改旧交易理由。
- 不自动生成纪律消息；本次只建立存储和展示容器。
- 不计算缺少连续账户净值支持的最大回撤或历史仓位比例。
- 不把教练字段写入只允许保存交易事实的账户底账数据库。

## 3. 信息架构

主导航调整为：

1. 今日
2. 训练时间线
3. 股票故事
4. 交易模式
5. 交易底账
6. 纪律规则

“交易模式”成为独立页面。训练时间线仍存在，但不再占用首页工作区。

## 4. 首页教练台

### 4.1 顶部账户事实与交易能力

顶部采用非对称两排布局，不使用 12 个等权方格。

左侧唯一主数字：

- 总盈亏（含当前持仓）
- 最新行情日期
- 费用已经进入成本口径的说明

右侧第一排账户事实：

- 已实现盈亏
- 持仓浮动盈亏
- 总费用
- 全历史交易股票数

交易股票数定义为账户底账中 `BUY` 或 `SELL` 成交涉及的不同六位证券代码数量；不包含逆回购、资金流水和证券现金调整。

右侧第二排为无独立外框的能力事实轨道，按语义合并为三组：

1. 样本与赢面：完整周期数、周期胜率
2. 盈利质量：平均盈亏比、利润因子
3. 周期结果与持有时间：单周期期望、平均持股自然日、中位持股自然日

当前底账快照用于验收，不作为永久常量：

| --- | ---: |
| 总盈亏（含持仓） | -58,618.30 元 |
| 已实现盈亏 | -45,430.00 元 |
| 持仓浮动盈亏 | -13,188.30 元 |
| 总费用 | 9,670.61 元 |
| 交易股票数 | 118 |
| 完整周期 | 204 |

| 单周期期望 | -229.92 元 |
| 平均持股自然日 | 13.75 天 |
| 中位持股自然日 | 3 天 |

### 4.2 教练资格区

资格采用两层结构：

1. 全局教练闸门
2. 具体模式资格

状态只使用文字，不使用容易被理解成买卖信号的红黄绿交通灯：

- `待核验`：关键市场、计划或风险数据缺失。
- `风险锁定`：存在红牌或未关闭风险，不增加新风险。
- `仅观察`：全局闸门通过，但没有模式满足触发条件。
- `可进入验证`：某个交易前已定义的模式条件完整，可形成验证样本；不代表必须交易。

首页左侧突出全局闸门的状态、原因和下一项必须核验的事实。右侧显示当前交易正在验证的模式摘要。

### 4.3 今日工作区

首页不再展示最近训练日列表。该区域改为三栏：

1. 今日交易预案
2. 今日股票池
3. 当前持仓

股票池展示完整 15 支候选，使用固定高度和内部滚动，避免扩大整页高度。每行至少展示股票、题材篮子和买点类型，并链接到完整研究池文档。

当前持仓保留数量、券商式成本、最新价、浮动盈亏和故事入口。

若目标交易日没有预案或股票池，回退到最近一份有效文件，并明确显示：

- 文件目标日期
- `可能过期` 状态
- 原始文档入口

不得自动生成补位内容。

### 4.4 交易纪律消息

当前持仓下方预留聊天消息样式的可滚动纪律区域。

本次不定义消息生产逻辑。页面只读取结构化状态中的生效消息；没有消息时显示“暂无已发布纪律消息”，不自动拼装或编造内容。

## 5. 交易周期统一口径

### 5.1 周期边界

- 一只股票从空仓后的首次买入开始一个周期。
- 持仓归零时周期结束。
- 同日清仓后重新买入仍属于同一个周期，保持券商式滚动成本连续性。
- 隔日重新买入开启新周期。
- 未清仓周期状态为 `open`，不进入闭环能力指标。

周期 ID 必须在进行中与已结束状态之间保持不变。它基于证券代码和周期首次买入成交的规范化事实生成确定性哈希；规范化事实至少包含日期、时间、方向、数量、价格和净发生金额。最终清仓信息不得进入 ID。这样周期从 `open` 变成 `closed` 时，模式样本关联不会失效。

### 5.2 持股天数

- 使用自然日。
- 从周期首次买入日期到最终清仓日期计算。
- 首页同时展示平均数与中位数，避免少数超长周期扭曲理解。
- 股票故事页逐周期展示持股自然日。

### 5.3 周期能力公式

能力指标只使用已结束周期，均为含交易费用口径：

- 周期胜率 = 盈利周期数 / 已结束周期数
- 平均盈亏比 = 平均盈利周期金额 / 平均亏损周期金额绝对值
- 利润因子 = 全部盈利周期盈利总额 / 全部亏损周期亏损绝对值
- 单周期期望 = 全部周期交易盈亏合计 / 已结束周期数

边界显示规则：

- 没有已结束周期时，四项能力指标均显示 `待积累`。
- 没有亏损周期时，平均盈亏比和利润因子显示 `无亏损样本`，不显示无穷大。
- 没有盈利周期时，平均盈亏比显示 `无盈利样本`，利润因子显示 `0.00`。

现金调整进入账户总盈亏和对应区间已实现盈亏，但不进入周期胜率、平均盈亏比、利润因子和单周期期望。

未清仓持仓只在顶部账户事实、当前持仓和进行中故事周期中展示。

## 6. 交易底账时间筛选

### 6.1 固定与变化区域

顶部当前账户事实始终使用最新完整账户口径，不随历史筛选改变。

时间筛选只更新：

- 区间统计
- 区间成交流水
- 区间单票结果

### 6.2 时间控件

支持以下粒度：

- 全部
- 日
- 周
- 月
- 年
- 自定义

`日 / 周 / 月 / 年` 是时间粒度导航。选择粒度后显示当前周期标签及前后箭头，可浏览任意历史周期。

- 周以周一至周日为日期范围。
- 月使用自然月。
- 年使用自然年。
- 自定义使用包含首尾日期的起止区间。
- 默认值为 `全部`。

筛选状态写入 URL，刷新和复制链接后保持一致。建议参数：

```text
?grain=month&period=2026-07
?grain=custom&from=2026-01-01&to=2026-07-10
```

无效日期、结束早于开始或超出底账范围时，不修改数据并显示字段级错误。

### 6.3 区间统计

区间统计至少包括：

- 区间已实现盈亏
- 区间完整周期数
- 区间周期胜率
- 区间利润因子
- 区间费用
- 区间交易股票数

完整周期按最终清仓日期归入日、周、月、年或自定义区间。跨区间周期不拆分盈亏。

成交、费用和证券现金调整按实际发生日期归入区间。

区间单票结果聚合清仓日期位于该区间的完整周期，并单独加入区间内对应证券现金调整。

## 7. 股票故事页

### 7.1 页面层级

顶部保留该股票全历史摘要：

- 总周期数
- 总盈亏
- 周期胜率
- 平均持股自然日
- 中位持股自然日
- 当前是否有进行中周期

主区默认聚焦：

- 当前持仓股票：进行中周期
- 已清仓股票：最近结束周期

其他周期通过周期列表切换，不一次铺开全部成交。

### 7.2 周期详情

每个周期至少展示：

- 首次买入、最后加仓、最终清仓日期
- 持股自然日
- 周期成本、卖出净收入、周期盈亏和周期收益率
- 周期内成交事件
- 关联模式
- 执行结果
- 模式证据方向
- 关联手记、预案和复盘

同一周期可以包含多次买入、减仓、做 T 和卖出事件，但只有一个周期财务结果。

## 8. 交易模式档案

### 8.1 独立页面

新增 `modes.html`，状态视图包括：

- 验证中
- 待评审
- 可复制
- 应避免

首页只显示当前验证模式摘要，并链接到模式详情。

### 8.2 模式定义

每个模式必须包含：

- 模式名称与版本
- 适用环境
- 触发条件
- 执行边界
- 失效条件
- 最大风险
- 下一次验证要求
- 当前状态

只有完整定义并在交易前绑定的模式，才能产生正式验证样本。

### 8.3 样本分类

每个样本分别记录：

- 执行结果：`planned`、`violated`、`insufficient`
- 财务结果：周期盈亏、收益率、持股自然日
- 证据方向：`support`、`oppose`、`indeterminate`
- 证据类型：`formal`、`historical_reference`

旧交易与规则启用前的当前周期可作为 `historical_reference` 展示，但不计入升级所需的正式样本。

至少 3 个交易前绑定、按计划执行且证据可判断的正式样本后，模式只进入人工评审，不自动升级为可复制。

达到样本门槛时，系统只派生 `review_ready: true`。模式仍保持 `validating`，直到人工把状态改为 `review`；从 `review` 到 `replicable` 或 `avoid` 同样只能人工确认。

## 9. 私有结构化状态

### 9.1 交易模式

新增 `state/trading_modes.json`：

```json
{
  "version": 1,
  "modes": [
    {
      "id": "risk-anchor-close",
      "name": "风险锚失效后的尾盘处理",
      "status": "validating",
      "version": "0.1",
      "applicable_environment": ["持仓风险锚在收盘前确认失效"],
      "trigger_conditions": ["风险锚失效", "可卖仓位已明确"],
      "execution_boundaries": ["只处理可卖风险", "T+1 锁定仓位单独披露"],
      "invalidation_conditions": ["盘后改变风险锚", "处理前新增仓位"],
      "max_risk": "执行前必须计算当日最低可降持仓",
      "next_validation_requirement": "下一次交易前绑定模式并记录完整触发条件",
      "samples": [
        {
          "cycle_id": "cyc_300260_20260610_4f8a0b1d",
          "evidence_type": "historical_reference",
          "execution_result": "insufficient",
          "evidence_direction": "indeterminate",
          "note": "规则启用前的进行中周期，只作为历史参考",
          "source_paths": ["reports/run_20260710_review/coach_note.md"]
        }
      ]
    }
  ]

```

派生数量如正式样本数、支持样本数和平均结果不重复存储，由生成器计算。

### 9.2 纪律消息

新增 `state/discipline_feed.json`，初始内容为：

```json
{
  "version": 1,
  "messages": []
}
```

消息字段预留：

- `id`
- `status`: `draft`、`active`、`archived`
- `level`: `reminder`、`red_card`
- `scope`: `global`、`stock`、`mode`
- `stock_code`
- `mode_id`
- `message`
- `source_path`
- `effective_at`
- `expires_at`
- `created_at`

页面只读取 `active` 消息。所有路径必须为项目内相对路径，不在公开站点数据中写入绝对路径。

## 10. 数据流与兼容性

1. `state/account_ledger.sqlite` 继续只保存交易事实。
2. 站点生成器从底账计算账户事实、交易周期、区间统计和股票故事。
3. `state/trading_modes.json` 通过稳定周期 ID 关联计算出的周期。
4. `state/discipline_feed.json` 提供首页纪律消息。
5. Markdown 手记、预案、股票池和状态文件继续作为教练叙事与证据来源。
6. 现有 `state/personal_trading_modes.md` 不删除、不自动迁移；经过人工审核的结构化模式才写入 JSON。
7. 生成后的 `reports/personal_site/site_data.json` 只包含页面所需字段，并继续处于 Git 忽略范围。

## 11. 页面状态与错误处理

- 行情缺失：总盈亏显示待核验，不使用陈旧价格静默计算。
- 预案或股票池回退：显示原始日期和可能过期。
- 股票池不足 15 支：展示实际数量，不补虚假候选。
- 无当前持仓：展示空状态，纪律消息仍可显示全局内容。
- 无生效纪律消息：显示“暂无已发布纪律消息”。
- 无正式模式样本：显示 `0 / 3`，历史参考单独计数。
- JSON 文件缺失：按空版本处理并显示空状态。
- JSON 结构错误：页面继续生成其他区域，并在对应模块显示“状态数据待修复”。
- 筛选无匹配：保留筛选控件，区间统计显示 0，表格显示空状态。

## 12. 测试与验收

### 12.1 单元测试

- 同日清仓回补保持同一周期。
- 隔日重新买入创建新周期。
- 未清仓周期不进入能力指标。
- 自然日持股天数、平均数和中位数计算正确。
- 胜率、平均盈亏比、利润因子和单周期期望公式正确。
- 现金调整不进入周期能力指标。
- 周期按清仓日期归入区间。
- 成交与费用按发生日期归入区间。
- 日、周、月、年和自定义日期边界正确。
- 旧周期只作为历史参考，不计入正式样本数。
- 预案与股票池缺失时按规则回退并标记过期。

### 12.2 集成测试

- 生成首页、模式页、底账页和周期故事页。
- 15 支股票池完整进入首页滚动区域。
- 当前账户事实不随底账筛选改变。
- 区间统计、流水和单票结果使用同一 URL 区间。
- 模式样本能通过周期 ID 连接到股票周期详情。
- 缺失或损坏的结构化状态不阻断整站生成。
- 所有站内链接存在，不回退到旧版 HTML 文档。

### 12.3 浏览器验收

- 桌面与 390px 手机视口无整页横向溢出。
- 金额符号与数值不拆行。
- 股票池与纪律消息在自身容器内滚动。
- 时间粒度、前后周期、自定义区间、搜索和分页均更新 URL。
- 浏览器前进、后退和刷新能恢复筛选状态。
- 键盘焦点可见，日期字段和按钮有可访问名称。

## 13. 实施边界

本轮实施只改个人站生成器、共享前端资源、私有结构化状态初始化和相关测试。不会自动生成纪律内容，不会修改账户底账交易事实，不会重写历史教练手记，也不会自动把旧交易升级为正式模式样本。
